Add transition_issue_label primitive on LabelsMixin

Introduce a config-free, atomic label transition primitive alongside
add_labels/remove_labels/set_labels. Computes
(current - labels_to_clear) | {new_label} and applies via set_labels.

- Idempotent short-circuit: if new_label is already present AND there
  is no overlap between current labels and labels_to_clear, returns
  True without calling set_labels.
- Guards against get_issue swallowing a failure via its own
  default_return: if get_issue returns an empty IssueData (number=0),
  primitive returns False without calling set_labels.
- ValueError from validate_issue_number and 401/403 GithubException
  propagate via the @_handle_github_errors decorator; other errors
  return False.

Adds TestTransitionIssueLabel test class (8 cases) covering basic
add+clear, idempotent no-op, stray-label cleanup, empty clear set,
invalid issue number, auth error reraise, set_labels failure, and
get_issue failure.

update_workflow_label is not modified in this step; the refactor to
delegate to the primitive comes in Step 3.
