Group CONFIG entries by [section] header in verify output (#824 step 4)

Replace the flat config-printing loop in `execute_verify` with a
grouping loop keyed on `entry["label"]`. Each `[section]` header is
emitted once, items are indented below it, and a blank line separates
groups.

For entries inside a `[section]`:
- Split value on first space -> `key` / `rest`.
- If the first token looks like a Python-identifier-like key
  (new `_looks_like_key` helper; rejects Python keywords such as
  `not` so that `not configured` is not rendered as a key column),
  render as `    {key:<18s} {symbol} {rest}`.
- Otherwise (e.g. `[OK] 6 repos configured`, `not configured`),
  render the whole value without a key column.

Top-level rows (`Config file`, `Expected path`, `Hint`, `Parse error`)
continue to render as two-space indented rows.

`verify_config()` return shape is unchanged.

Adds `TestConfigGrouping` covering header-once grouping, indentation,
blank-line separation, key extraction, the `Config file` top-level
row, info entries without a clean `key value` split, and summary
entries whose value begins with a status symbol. Also adds
`TestLooksLikeKey` covering the identifier/keyword/bracket rules.
