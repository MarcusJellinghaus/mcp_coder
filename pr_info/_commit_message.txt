Add ENVIRONMENT section to verify output (#824 step 3)

Introduce `_print_environment_section()` in `verify.py` and invoke it
at the top of `execute_verify`. The section prints Python runtime info
(version, executable, virtualenv, PYTHONPATH) followed by the installed
versions of the four mcp-coder packages. Missing packages render as
`[ERR] not installed`.

Package list is lifted to a module-level `_ENVIRONMENT_PACKAGES` tuple
for testability. Virtualenv detection uses `sys.prefix != sys.base_prefix`
and PYTHONPATH falls back to `(not set)` when missing or empty.

Adds `TestEnvironmentSection` covering the header, Python version row,
missing PYTHONPATH, missing package, and non-venv cases.
