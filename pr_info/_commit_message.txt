Add MCP config warnings section and log filter to verify (#824 step 5)

Parse `.mcp.json` for unresolved `${...}` placeholders in
`mcpServers[*].env[*]` values and render them in a new
`=== MCP CONFIG WARNINGS ===` section. The section is omitted when
there are no findings or when `.mcp.json` is missing/invalid.

Also suppress the raw `WARNING: env['PYTHONPATH'] contains unexpanded
variable reference` stdout noise from `langchain_mcp_adapters` via a
scoped `logging.Filter` installed around the `verify_mcp_servers()`
call. A try/finally ensures the filter is removed afterwards.

Changes:
- `src/mcp_coder/cli/commands/verify.py`
  - Add `import json`.
  - Add `_DropUnexpandedWarnings` logging.Filter subclass that drops
    records whose message contains "unexpanded variable".
  - Add `_collect_mcp_warnings(path)` parser helper:
    - Returns `[]` for `None`, missing file, or invalid JSON.
    - Emits one `"server / env_var  value"` row per env value that
      contains any `${...}` placeholder, preserving the full original
      value (including suffixes like `/src` or `\Lib\`).
  - Wrap the `verify_mcp_servers()` call with the scoped log filter
    (try/finally).
  - Render the new section immediately after the langchain-MCP
    server list and before the MCP edit smoke test / test-prompt rows.

- `tests/cli/commands/test_verify_orchestration.py`
  - Add `TestMcpConfigWarnings` covering parser edge cases (None,
    missing file, invalid JSON, no placeholders, single and multiple
    findings) and the orchestrator behavior (section omitted when
    clean, rendered when findings exist, log filter suppresses the
    `unexpanded variable` warning and is removed after the call).
  - Add `TestDropUnexpandedFilter` covering the filter itself.
