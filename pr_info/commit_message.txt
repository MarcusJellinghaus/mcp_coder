feat(icoder): auto-store sessions for --continue-session support (#765)

Add `provider` property to LLMService protocol and both implementations
(RealLLMService, FakeLLMService) so AppCore can create a ResponseAssembler.

Update AppCore.stream_llm() to accumulate stream events via
ResponseAssembler and call store_session() after streaming completes,
mirroring the prompt command's pattern. This enables --continue-session
to find prior iCoder sessions for the Claude provider.

Fix ErrorAfterChunksLLMService test helper to satisfy updated protocol
(add provider property and reset_session method).
