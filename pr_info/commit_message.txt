fix: add commands to error statuses so Claude session launches after env setup

Five error statuses (planning_failed, implementing_failed, ci_fix_needed,
llm_timeout, pr_creating_failed) had vscodeclaude config but no commands,
causing startup scripts to exit without launching a Claude session.

Add "commands": ["/check_branch_status"] to all five error statuses in
labels.json. Add parametrized tests verifying error statuses have commands
and that pr_created intentionally does not.
