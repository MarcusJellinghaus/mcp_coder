feat(tui): add Windows CMD codepage auto-fix to TuiChecker

Add _check_windows_cmd_codepage() that detects non-UTF-8 console
codepage on Windows via ctypes.windll.kernel32.GetConsoleOutputCP(),
silently fixes it to 65001 (UTF-8) with SetConsoleOutputCP(), and
registers an atexit handler to restore the original codepage.

Wired into run_all_checks() before warning checks so silent fixes
are collected and applied first.

Tests mock ctypes.windll so they run on all platforms:
- Non-UTF-8 codepage detection
- Already-UTF-8 no-op
- Wrong platform no-op
- Fix function calls SetConsoleOutputCP(65001)
- Atexit restore registered with original codepage
- Silent fix logged through run_all_checks()
