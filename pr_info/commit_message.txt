feat(llm): thread tools param through prompt_llm_stream (#741)

Add optional `tools: list[Any] | None` parameter to `prompt_llm_stream()`
in the high-level LLM interface. The parameter is forwarded to
`ask_langchain_stream()` on the langchain path; the claude path ignores it.

This is part of threading persistent MCP tools through the provider stack
so that `RealLLMService` can pass pre-built tools from `MCPManager`.

Changes:
- src/mcp_coder/llm/interface.py: add `tools` param to `prompt_llm_stream()`,
  forward to `ask_langchain_stream()`
- tests/llm/test_interface.py: add tests for tools forwarding (langchain),
  tools=None default, and claude path ignoring tools; update existing
  langchain stream routing test for new kwarg
