Step 2: wrap MCP launch failures as LLMMCPLaunchError in agent (#830)

In both run_agent and run_agent_stream (tools=None branch), wrap
FileNotFoundError / PermissionError raised on entering
client.session(server_name) as LLMMCPLaunchError. The re-raised error
includes the server name, resolved command path, and original exception
class name so the CLI top-level handlers surface actionable diagnostics
instead of generic WinError 2 strings.

The try/except is placed inside the per-server loop so the server name
and resolved command are in scope, and covers the async context manager
entry itself (where stdio_client.__aenter__ raises FileNotFoundError /
PermissionError on Windows / POSIX).

LLMMCPLaunchError inherits from Exception (not ConnectionError) and is
not in CONNECTION_ERRORS, so _handle_provider_error in __init__.py
passes it through unchanged; the CLI top-level handlers print str(exc).

Tests added in tests/llm/providers/langchain/test_langchain_agent.py:
- run_agent wraps FileNotFoundError as LLMMCPLaunchError
- run_agent wraps PermissionError as LLMMCPLaunchError
- run_agent_stream wraps FileNotFoundError on first iteration
- run_agent_stream with pre-built tools does not construct
  MultiServerMCPClient (wrap path is skipped)
