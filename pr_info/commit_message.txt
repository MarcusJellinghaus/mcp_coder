feat(llm): add MCPManager for persistent MCP connections (#741)

Add MCPManager class that owns persistent MCP server connections for
the app's lifetime. Uses a dedicated daemon thread with its own asyncio
event loop. Connects lazily on first tools() call and caches results.
Recreates the client on failure for robustness.

New files:
- src/mcp_coder/llm/mcp_manager.py: MCPManager class + MCPServerStatus
- tests/llm/test_mcp_manager.py: 8 unit tests covering tools caching,
  lazy connect, status, close, error recovery, and empty config
