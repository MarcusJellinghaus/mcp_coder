Rewrite _render_tool_output() to use generic rendering (#820)

Remove field-filtering logic (_ENVELOPE_FIELDS, _MAIN_CONTENT_KEYS) and
the obsolete _render_value() helper. _render_tool_output() now only
unwraps the {"result": ...} envelope and delegates all JSON rendering
to _render_output_value() (added in step 1), which correctly splits
multiline strings like diffs.

Add a full=False parameter to _render_tool_output(); when True,
head/tail truncation is skipped. The StreamEventRenderer continues
using the default compact mode.

Rewrite the affected test classes: drop TestRenderToolOutputFieldFiltering,
add coverage for result-envelope unwrap (scalar, dict with multiline diff,
with extras), non-unwrap of text/content keys, and full-mode truncation
skip.
