feat(llm): add mcp_manager to RealLLMService (#741)

Add optional `mcp_manager` parameter to `RealLLMService` so persistent
MCP tools are threaded through to `prompt_llm_stream()`.

- Import `MCPManager` under `TYPE_CHECKING` to avoid circular imports
- In `stream()`, resolve tools from mcp_manager when present, pass as
  `tools=` kwarg to `prompt_llm_stream()`
- When no mcp_manager is set, `tools=None` preserves existing behavior
- Add two tests: with manager (tools forwarded) and without (None)
