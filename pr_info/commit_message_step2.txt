feat(coordinator): add watchdog set-status to coordinator templates for silent-death recovery (#713)

Update all six coordinator Jenkins command templates (Linux and Windows)
with exit code capture, a watchdog `gh-tool set-status` line using
`--from-status`, and explicit exit with the captured RC.

The watchdog runs unconditionally after the main mcp-coder command. It
only transitions the issue label if it is still at the in-progress
status (via --from-status precondition), rescuing issues stuck by a
silent process death where Python cleanup never ran.

Template changes:
- CREATE_PLAN_COMMAND_TEMPLATE / WINDOWS: rescue from status-03:planning
  to status-03f:planning-failed (passes --issue {issue_number})
- IMPLEMENT_COMMAND_TEMPLATE / WINDOWS: rescue from status-06:implementing
  to status-06f:implementing-failed
- CREATE_PR_COMMAND_TEMPLATE / WINDOWS: rescue from status-09:pr-creating
  to status-09f:pr-creating-failed

Add inline recovery matrix comment block documenting the four exit paths
and their watchdog behavior.

Add TestTemplateWatchdogLines test class (10 tests) covering exit code
capture, watchdog lines, RC exit, issue flag presence, placeholder
resolution, and recovery matrix comment.
